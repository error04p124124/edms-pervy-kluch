## Проверка доступа к страницам

### Страницы доступные всем ролям:
- `/` - Главная
- `/dashboard/` - Панель управления (с разным контентом по ролям)
- `/profile/` - Профиль пользователя

### Страницы только для admin и clerk:
- `/documents/create/` - Создание документа (ОБНОВЛЕН современный дизайн)
- `/documents/<id>/edit/` - Редактирование документа (ОБНОВЛЕН современный дизайн)
- `/documents/<id>/delete/` - Удаление документа
- `/templates/` - Список шаблонов (ОБНОВЛЕН современный дизайн)
- `/templates/create/` - Создание шаблона (ОБНОВЛЕН современный дизайн)
- `/templates/<id>/edit/` - Редактирование шаблона (ОБНОВЛЕН современный дизайн)
- `/templates/<id>/delete/` - Удаление шаблона
- `/requests/` - Обработка запросов

### Страницы только для admin и manager:
- `/team-stats/` - Статистика команды
- `/documents/<id>/approve/` - Утверждение документа

### Страницы для всех кроме employee:
- `/documents/` - Полный список документов (employee видит только свои)
- `/reports/` - Отчеты

## Технические детали

### Миксины доступа в documents/views.py:
- `ClerkRequiredMixin` - для admin и clerk
- `ManagerRequiredMixin` - для admin и manager
- `RoleRequiredMixin` - базовый класс, admin всегда проходит проверку

### Функции проверки доступа в documents/permissions.py:
- `can_create_document(user)` - admin, clerk
- `can_edit_document(user, document)` - admin, clerk
- `can_delete_document(user, document)` - admin, clerk
- `can_approve_document(user, document)` - admin, manager
- `can_manage_templates(user)` - admin, clerk
- `can_view_all_documents(user)` - admin, clerk, manager

### Все функции автоматически дают доступ администратору!
