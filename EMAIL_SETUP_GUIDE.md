# Настройка Email-уведомлений

## Конфигурация завершена ✅

Email-уведомления настроены и будут отправляться на адреса пользователей при следующих событиях:

### 1. **Назначение документа**
- **Когда:** Документ назначен ответственному
- **Кому:** Ответственному пользователю
- **Содержание:** "Вам назначен документ [название]"

### 2. **Согласование документа**
- **Когда:** Документ отправлен на согласование
- **Кому:** Следующему согласующему в маршруте
- **Содержание:** "Документ [название] ожидает вашего согласования"

### 3. **Одобрение документа**
- **Когда:** Документ одобрен на этапе или полностью
- **Кому:** Создателю документа
- **Содержание:** "Ваш документ [название] был одобрен"

### 4. **Отклонение документа**
- **Когда:** Документ отклонен
- **Кому:** Создателю документа
- **Содержание:** "Ваш документ [название] был отклонен" + комментарий

### 5. **Новое сообщение в чате**
- **Когда:** Пользователь получил сообщение
- **Кому:** Получателю сообщения
- **Содержание:** Первые 200 символов сообщения

### 6. **Регистрация документа**
- **Когда:** Документ зарегистрирован с назначением ответственного
- **Кому:** Ответственному
- **Содержание:** "Вам назначен документ [название]"

## Настройки Email

Используется Gmail SMTP:
- **Сервер:** smtp.gmail.com
- **Порт:** 587
- **TLS:** Включен
- **Отправитель:** error04p@gmail.com

## Формат писем

Все письма отправляются в HTML-формате с оформлением:
- Заголовок "ЭДО Первый ключ"
- Цветовая схема фиолетовая (#667eea)
- Содержит имя получателя
- Информация о документе (если применимо)
- Ссылка на систему
- Примечание о автоматическом уведомлении

## Требования к пользователям

⚠️ **Важно:** Email-уведомления отправляются только пользователям, у которых указан email в профиле.

### Как проверить/добавить email:

1. Перейти в профиль пользователя
2. Убедиться, что поле "Email" заполнено
3. Сохранить изменения

## Тестирование

### Шаг 1: Проверка email в профиле
```
1. Войти в систему
2. Перейти в Профиль
3. Убедиться, что email указан
```

### Шаг 2: Тест назначения документа
```
1. Создать новый документ из шаблона
2. Указать ответственного (у которого есть email)
3. Указать срок исполнения
4. Создать документ
5. Проверить почту ответственного
```

### Шаг 3: Тест чата
```
1. Открыть чат
2. Отправить сообщение пользователю
3. Проверить почту получателя
```

### Шаг 4: Тест согласования
```
1. Создать документ с маршрутом согласования
2. Отправить на согласование
3. Проверить почту первого согласующего
4. Одобрить этап
5. Проверить почту следующего согласующего
```

   - Убедиться, что создан пароль приложения

2. **Проверить логи Django:**

## Безопасность

✅ Пароль приложения хранится в настройках (не в коде)
✅ Используется TLS шифрование
✅ Email отправляется только зарегистрированным пользователям
✅ Автоматические письма помечены как "не отвечать"

## Важные замечания

- Email отправляется асинхронно (fail_silently=True), ошибки не блокируют работу системы
- Если email не указан, уведомление все равно создается в системе (колокольчик)
- Письма отправляются от имени ЭДО "Первый ключ"
- В теме письма всегда префикс [ЭДО]
