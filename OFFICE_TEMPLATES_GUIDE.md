# Обновление системы шаблонов - Работа с Office документами

## Что изменилось

Система шаблонов переделана для работы с реальными файлами Office форматов:
- **Word** (.docx) - документы, приказы, письма
- **Excel** (.xlsx) - таблицы, отчеты, реестры  
- **PowerPoint** (.pptx) - презентации
- **PDF** (.pdf) - документы для печати
- **HTML** (.html) - веб-документы (оставлено для совместимости)

## Как создать шаблон

### 1. Подготовка файла-шаблона

#### Word документ (.docx):
1. Откройте Microsoft Word
2. Создайте документ с нужной структурой
3. Используйте плейсхолдеры в фигурных скобках:
   ```
   ПРИКАЗ
   от {{дата}}
   
   Приказываю принять на работу {{имя}}
   на должность {{должность}} в {{отдел}}.
   
   Руководитель _____________
   ```
4. Сохраните как .docx

#### Excel таблица (.xlsx):
1. Откройте Microsoft Excel
2. Создайте таблицу с плейсхолдерами в ячейках:
   | Поле | Значение |
   |------|----------|
   | ФИО | {{имя}} |
   | Должность | {{должность}} |
   | Отдел | {{отдел}} |
   | Дата | {{дата}} |
3. Сохраните как .xlsx

#### PowerPoint презентация (.pptx):
1. Откройте Microsoft PowerPoint
2. Создайте слайды с плейсхолдерами в текстовых блоках
3. Сохраните как .pptx

### 2. Загрузка шаблона в систему

1. Перейдите в **Шаблоны** → **Создать шаблон**
2. Заполните форму:
   - **Название**: название шаблона
   - **Тип документа**: выберите тип (приказ, договор и т.д.)
   - **Формат файла**: выберите формат (docx, xlsx, pptx)
   - **Файл шаблона**: загрузите подготовленный файл
   - **Описание плейсхолдеров** (опционально): JSON с описаниями
     ```json
     {
       "имя": "ФИО сотрудника",
       "должность": "Должность",
       "отдел": "Отдел/подразделение",
       "дата": "Текущая дата (автоматически)"
     }
     ```
3. Нажмите **Сохранить**

### 3. Создание документа из шаблона

1. Перейдите в **Документы** → **Создать из шаблона**
2. Выберите шаблон из списка
3. Заполните поля:
   - **Название документа**
   - **ФИО** (для плейсхолдера {{имя}})
   - **Должность** (для плейсхолдера {{должность}})
   - **Отдел** (для плейсхолдера {{отдел}})
   - **Дополнительные поля** (для {{custom1}}, {{custom2}})
   - **Ответственный** и **Срок исполнения**
4. Нажмите **Создать**

Система автоматически:
- Создаст документ
- Заменит все плейсхолдеры на указанные значения
- Сгенерирует файл в нужном формате (.docx, .xlsx и т.д.)
- Сохранит файл в папку `media/generated/`

### 4. Скачивание сгенерированного документа

1. Откройте карточку документа
2. В разделе **"Сгенерированный документ"** нажмите кнопку скачивания
3. Файл загрузится на ваш компьютер в оригинальном формате

## Доступные плейсхолдеры

| Плейсхолдер | Описание | Автозаполнение |
|-------------|----------|----------------|
| `{{имя}}` | ФИО сотрудника | Вводится пользователем |
| `{{название}}` | Название документа | Из поля "Название" |
| `{{должность}}` | Должность | Вводится пользователем |
| `{{отдел}}` | Отдел/подразделение | Вводится пользователем |
| `{{дата}}` | Текущая дата | Автоматически (ДД.ММ.ГГГГ) |
| `{{время}}` | Текущее время | Автоматически (ЧЧ:ММ) |
| `{{custom1}}` | Дополнительное поле 1 | Вводится пользователем |
| `{{custom2}}` | Дополнительное поле 2 | Вводится пользователем |

## Примеры использования

### Пример 1: Приказ о приеме на работу (Word)
**Шаблон (приказ.docx):**
```
ПРИКАЗ № _______
от {{дата}}

О приеме на работу

ПРИКАЗЫВАЮ:

1. Принять на работу {{имя}} на должность {{должность}} 
   в {{отдел}} с {{дата}}.

2. Установить оклад согласно штатному расписанию.

3. Контроль за исполнением приказа возложить на {{custom1}}.

Руководитель                    ______________

```

**При создании документа:**
- имя = "Иванов Иван Иванович"
- должность = "Менеджер по продажам"
- отдел = "Отдел продаж"
- custom1 = "Петрова А.С."

**Результат:** Файл `Приказ_о_приеме_123.docx` со всеми заполненными данными

### Пример 2: Табель учета (Excel)
**Шаблон (табель.xlsx):**
| ФИО | Должность | Отдел | Дата |
|-----|-----------|-------|------|
| {{имя}} | {{должность}} | {{отдел}} | {{дата}} |

**Результат:** Файл `Табель_учета_124.xlsx` с заполненными ячейками

## Устранение неполадок

### Ошибка: "Библиотека python-docx не установлена"
**Решение:** Установите библиотеки:
```powershell
pip install python-docx openpyxl python-pptx
```

### Ошибка: "Файл шаблона не найден"
**Решение:** 
1. Проверьте, что файл загружен в админке
2. Убедитесь, что путь к media папке настроен в settings.py
3. Проверьте права доступа к папке media/templates/

### Плейсхолдеры не заменяются
**Проблема:** В Word документе плейсхолдеры разбиты на несколько фрагментов

**Решение:**
1. Удалите плейсхолдер
2. Напишите его заново одним символом (не копируйте)
3. Используйте простой текст без форматирования

### Файл не генерируется
**Проверьте:**
1. Формат файла соответствует расширению (docx для Word, xlsx для Excel)
2. В шаблоне нет поврежденных элементов
3. Есть права на запись в папку media/generated/

## API для генерации (для разработчиков)

```python
from documents.office_utils import generate_document_from_template

# Генерация Word документа
success, error = generate_document_from_template(
    template_file_path='/path/to/template.docx',
    template_format='docx',
    output_path='/path/to/output.docx',
    replacements={
        'имя': 'Иванов И.И.',
        'должность': 'Менеджер',
        'дата': '20.01.2026'
    }
)

if success:
    print("Документ успешно сгенерирован!")
else:
    print(f"Ошибка: {error}")
```

## Преимущества нового подхода

✅ **Реальные файлы Office** - пользователи получают .docx, .xlsx файлы, которые можно редактировать  
✅ **Форматирование сохраняется** - стили, шрифты, таблицы из шаблона остаются  
✅ **Совместимость** - файлы открываются в Microsoft Office, LibreOffice, Google Docs  
✅ **Автоматизация** - не нужно вручную заполнять документы  
✅ **История** - все действия логируются в DocumentHistory  
✅ **Безопасность** - контроль доступа к скачиванию файлов

## Обратная совместимость

Старые HTML шаблоны продолжают работать:
- Если `file_format='html'` и заполнен `html_template`, используется старая логика
- Существующие документы не затронуты
- При необходимости можно мигрировать старые шаблоны на новые форматы
