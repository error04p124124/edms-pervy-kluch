# Руководство по динамическим плейсхолдерам

## Обзор

Система динамических плейсхолдеров позволяет администраторам создавать шаблоны документов с настраиваемыми полями. Больше не нужно использовать только предопределенные поля - теперь каждый шаблон может иметь свой уникальный набор плейсхолдеров.

## Как это работает

### 1. Создание шаблона с плейсхолдерами

При создании или редактировании шаблона документа:

1. **Перейдите в раздел "Управление плейсхолдерами"**
   - Здесь вы увидите автоматически доступные плейсхолдеры:
     - `{{дата}}` - текущая дата
     - `{{время}}` - текущее время
     - `{{название}}` - название документа

2. **Добавьте свои плейсхолдеры**
   - Нажмите кнопку "Добавить плейсхолдер"
   - Заполните форму в модальном окне:

**Поля модального окна:**

- **Ключ плейсхолдера** (обязательно)
  - Это имя переменной в шаблоне
  - Используйте только буквы, цифры и подчеркивание
  - Примеры: `имя_сотрудника`, `адрес`, `телефон`, `inn`
  - В шаблоне будет использоваться как `{{ключ}}`

- **Отображаемое название** (обязательно)
  - Текст, который увидит пользователь при заполнении документа
  - Примеры: "ФИО сотрудника", "Адрес проживания", "Номер телефона"

- **Тип поля** (обязательно)
  - `text` - обычное текстовое поле (одна строка)
  - `textarea` - многострочное текстовое поле
  - `date` - поле для выбора даты
  - `number` - поле для ввода чисел

- **Текст подсказки** (опционально)
  - Вспомогательный текст под полем
  - Примеры: "Введите полный адрес с индексом", "Формат: +7 (999) 999-99-99"

- **Значение по умолчанию** (опционально)
  - Предзаполненное значение поля
  - Пользователь может его изменить
  - Примеры: "Не указано", "+7", "г. Москва"

- **Обязательное поле** (галочка)
  - Если включено - пользователь обязан заполнить это поле
  - Если выключено - поле можно оставить пустым

### 2. Использование плейсхолдеров в файле шаблона

#### Для Word документов (.docx):

```
Договор №{{номер_договора}} от {{дата}}

Настоящий договор заключен между:

Заказчик: {{имя_заказчика}}
Адрес: {{адрес_заказчика}}
ИНН: {{инн_заказчика}}

Исполнитель: {{название}}
Дата заключения: {{дата}}
```

#### Для Excel таблиц (.xlsx):

| Поле | Значение |
|------|----------|
| ФИО | {{фио_сотрудника}} |
| Должность | {{должность}} |
| Отдел | {{отдел}} |
| Дата приема | {{дата_приема}} |

#### Для PowerPoint презентаций (.pptx):

```
Слайд 1:
Отчет за {{период}}
Подготовил: {{автор}}

Слайд 2:
Статистика на {{дата}}
Показатель: {{показатель}}
```

### 3. Создание документа из шаблона

Когда пользователь создает документ из вашего шаблона:

1. Выбирает шаблон из списка
2. Страница обновляется и показывает:
   - Стандартные поля (название, ответственный, срок)
   - **Ваши кастомные поля** с названиями и подсказками
3. Заполняет все поля
4. Нажимает "Создать документ"
5. Система генерирует файл, заменяя все `{{плейсхолдеры}}` на введенные значения

## Примеры использования

### Пример 1: Трудовой договор

**Плейсхолдеры:**
- `фио_работника` (text, обязательное) - "ФИО работника"
- `должность` (text, обязательное) - "Должность"
- `оклад` (number, обязательное) - "Оклад (руб.)"
- `дата_начала` (date, обязательное) - "Дата начала работы"
- `адрес_регистрации` (textarea, обязательное) - "Адрес регистрации"
- `паспорт` (text, обязательное) - "Паспортные данные"

**В Word шаблоне:**
```
ТРУДОВОЙ ДОГОВОР №____ от {{дата}}

Работодатель: ООО "Первый ключ"
Работник: {{фио_работника}}
Паспорт: {{паспорт}}
Адрес: {{адрес_регистрации}}

Работник принимается на должность {{должность}}
с окладом {{оклад}} рублей в месяц.
Дата начала работы: {{дата_начала}}.
```

### Пример 2: Акт приема-передачи

**Плейсхолдеры:**
- `номер_акта` (text, обязательное, по умолчанию: "Б/Н") - "Номер акта"
- `сторона1` (text, обязательное) - "Передающая сторона"
- `сторона2` (text, обязательное) - "Принимающая сторона"
- `описание_товара` (textarea, обязательное) - "Описание передаваемого"
- `количество` (number, обязательное) - "Количество"
- `примечание` (textarea) - "Примечания"

### Пример 3: Командировочное удостоверение

**Плейсхолдеры:**
- `фио` (text, обязательное) - "ФИО сотрудника"
- `табельный_номер` (text, обязательное) - "Табельный номер"
- `место_командировки` (text, обязательное) - "Место командировки"
- `цель` (textarea, обязательное) - "Цель командировки"
- `дата_начала` (date, обязательное) - "Дата начала"
- `дата_окончания` (date, обязательное) - "Дата окончания"
- `транспорт` (text, по умолчанию: "Не указано") - "Вид транспорта"

## Советы и рекомендации

### ✅ Хорошие практики:

1. **Используйте понятные ключи**
   - Хорошо: `фио_директора`, `номер_счета`, `срок_действия`
   - Плохо: `field1`, `x`, `temp`

2. **Давайте подробные названия**
   - Хорошо: "ФИО руководителя организации"
   - Плохо: "Имя"

3. **Добавляйте подсказки для сложных полей**
   - Для паспорта: "Формат: 1234 567890"
   - Для телефона: "Формат: +7 (999) 999-99-99"
   - Для ИНН: "10 или 12 цифр"

4. **Используйте правильный тип поля**
   - Для адресов → `textarea`
   - Для дат → `date` (автоматически форматируется как ДД.ММ.ГГГГ)
   - Для сумм → `number`
   - Для кратких данных → `text`

5. **Отмечайте обязательные поля**
   - Критичные данные (ФИО, номера документов) → обязательные
   - Необязательную информацию → необязательные

### ❌ Чего избегать:

1. **Не используйте спецсимволы в ключах**
   - Избегайте: пробелы, дефисы, точки, кавычки
   - Используйте: буквы, цифры, подчеркивания

2. **Не дублируйте автоматические плейсхолдеры**
   - Не создавайте свой `дата` - он уже есть
   - Не создавайте свой `название` - он уже есть

3. **Не создавайте слишком много полей**
   - Оптимально: 5-10 полей
   - Если нужно больше - подумайте о разделении на несколько шаблонов

## Технические детали

### Форматы плейсхолдеров в файлах:

Система поддерживает несколько форматов записи:
- `{{ключ}}` - основной формат (двойные фигурные скобки)
- `{ключ}` - одинарные фигурные скобки
- `[ключ]` - квадратные скобки

Все три формата будут заменены на значение поля.

### Автоматическое форматирование:

- Поля типа `date` автоматически форматируются как `ДД.ММ.ГГГГ`
- Пустые поля заменяются на пустую строку
- Числовые поля преобразуются в строку

### Префикс полей в форме:

Технически все кастомные поля в форме имеют префикс `placeholder_`:
- Ключ `фио` → поле формы `placeholder_фио`
- Ключ `адрес` → поле формы `placeholder_адрес`

Это делается автоматически, вам не нужно об этом беспокоиться.

## Устранение проблем

### Проблема: Плейсхолдер не заменяется в документе

**Решение:**
- Проверьте, что ключ в шаблоне точно совпадает с ключом в настройках
- Проверьте формат записи: `{{ключ}}`, `{ключ}` или `[ключ]`
- Убедитесь, что в ключе нет опечаток

### Проблема: Не вижу свои поля при создании документа

**Решение:**
- После выбора шаблона страница должна обновиться
- Если не обновилась - обновите страницу вручную (F5)
- Проверьте, что плейсхолдеры сохранены в шаблоне

### Проблема: Поле обязательное, но пустое значение проходит

**Решение:**
- Убедитесь, что галочка "Обязательное поле" установлена
- Пересохраните шаблон
- Очистите кэш браузера

### Проблема: Дата отображается в неправильном формате

**Решение:**
- Убедитесь, что тип поля установлен как `date`
- Система автоматически форматирует даты как ДД.ММ.ГГГГ

## Миграция со старых шаблонов

Если у вас были старые шаблоны с жестко заданными полями (`{{имя}}`, `{{должность}}` и т.д.), вы можете:

1. **Создать новый шаблон с нужными плейсхолдерами**
2. **Скопировать файл шаблона из старого в новый**
3. **Настроить плейсхолдеры в новом шаблоне**
4. **Использовать новый шаблон вместо старого**

Старые шаблоны будут продолжать работать, но не будут иметь динамических полей.

## Заключение

Система динамических плейсхолдеров дает вам полный контроль над структурой шаблонов документов. Вы можете создавать шаблоны для любых типов документов с любым набором полей, без необходимости изменять код системы.

Если у вас есть вопросы или предложения - обращайтесь к администратору системы.
