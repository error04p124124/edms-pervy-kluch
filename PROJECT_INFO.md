# Система электронного документооборота "Первый ключ"

## 📋 Описание проекта

Полнофункциональная система электронного документооборота (ЭДО) для автоматизации работы делопроизводителя в компании ООО "Первый ключ".

### Ключевые особенности:
- ✅ Автоматическая регистрация и нумерация документов (формат ГГГГ/ММ/НННН)
- ✅ **Создание документов из Office шаблонов (Word, Excel, PowerPoint)**
- ✅ **Автоматическая генерация файлов .docx, .xlsx, .pptx с заменой плейсхолдеров**
- ✅ **Скачивание сгенерированных документов в оригинальном формате**
- ✅ Гибкая система маршрутизации и согласования
- ✅ Контроль сроков с подсветкой просроченных документов
- ✅ Полная история изменений каждого документа
- ✅ Система уведомлений и чат между пользователями
- ✅ Отчеты и аналитика с печатью
- ✅ Современный веб-интерфейс

## 🎯 Целевая аудитория

1. **Делопроизводители** - основная роль с полным доступом
2. **Руководители** - согласование и утверждение документов
3. **Сотрудники** - создание заявок и отслеживание статусов
4. **Администраторы** - управление системой

## 🛠 Технический стек

### Backend
- Django 5.1.4
- Python 3.10+
- SQLite (можно заменить на PostgreSQL/MySQL)

### Frontend
- Bootstrap 5 (современный дизайн)
- Font Awesome (иконки)
- Chart.js (графики и аналитика)
- Vanilla JavaScript (без jQuery)

### Дополнительные библиотеки
- django-crispy-forms + crispy-bootstrap5 (красивые формы)
- **python-docx (работа с Word .docx)**
- **openpyxl (работа с Excel .xlsx)**
- **python-pptx (работа с PowerPoint .pptx)**
- WeasyPrint (экспорт в PDF)
- python-dotenv (управление конфигурацией)
- Pillow (обработка изображений)

## 📁 Структура базы данных

### Основные модели:

**UserProfile**
- Расширение User с ролями
- Роли: clerk, manager, employee, admin
- Отдел, должность, контакты

**Document**
- Основная модель документа
- Автоматическая нумерация
- Статусы: draft, in_review, approved, rejected, archived
- Сроки исполнения

**DocumentTemplate**
- Шаблоны для генерации документов
- Типы: приказ, договор, акт, письмо и др.
- **Форматы: Word (.docx), Excel (.xlsx), PowerPoint (.pptx), HTML**
- **Поддержка плейсхолдеров {{имя}}, {{должность}}, {{дата}} и др.**
- **Автоматическая замена плейсхолдеров при генерации**

**DocumentHistory**
- Полный аудит изменений
- Кто, что, когда

**Notification**
- Система уведомлений пользователей

**WorkflowStep**
- Этапы маршрута согласования

## 🔧 Настройка для production

1. Измените SECRET_KEY в .env
2. Установите DEBUG=False
3. Настройте ALLOWED_HOSTS
4. Используйте PostgreSQL вместо SQLite
5. Настройте веб-сервер (nginx + gunicorn)
6. Настройте статику через whitenoise или CDN
7. Включите HTTPS
8. Настройте резервное копирование
