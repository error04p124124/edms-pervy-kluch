```

   │ Добавляет плейсхолдеры через модальное окно │

   │ {                                           │
   │   "фио": {                                  │
   │     "label": "ФИО",                         │
   │     "type": "text",                         │
   │     "required": true,                       │

   │   }                                         │
   │ }                                           │
   └─────────────────────────────────────────────┘
                     ↓
   ┌─────────────────────────────────────────────┐
   │ DocumentTemplate.placeholders = JSON        │
   │ Сохраняется в базе данных                   │
   └─────────────────────────────────────────────┘

2. СОЗДАНИЕ ДОКУМЕНТА:
   ┌─────────────────────────────────────────────┐
   │ Пользователь выбирает шаблон                │
   │ ?template=X в URL                           │
   └─────────────────────────────────────────────┘

   ┌─────────────────────────────────────────────┐
   │ DocumentFromTemplateForm.__init__()         │
   │ Читает template.placeholders                │
   │ Создает динамические поля:                  │
   │   self.fields['placeholder_фио'] =          │
   │     CharField(label="ФИО", required=True)   │
   └─────────────────────────────────────────────┘
                     ↓
   ┌─────────────────────────────────────────────┐
   │ Django рендерит форму                       │
   │ JavaScript группирует placeholder_ поля     │
   │ Пользователь видит красиво оформленную форму│
   └─────────────────────────────────────────────┘
                     ↓
   ┌─────────────────────────────────────────────┐
   │ Пользователь заполняет и отправляет         │
   │ POST данные: placeholder_фио="Иванов..."    │
   └─────────────────────────────────────────────┘
                     ↓
   ┌─────────────────────────────────────────────┐
   │ create_from_template() view                 │
   │ Извлекает значения:                         │
   │   for key in template.placeholders:         │
   │     value = form.cleaned_data[              │
   │       f'placeholder_{key}']                 │
   │   replacements[key] = value                 │
   └─────────────────────────────────────────────┘
                     ↓
   ┌─────────────────────────────────────────────┐
   │ generate_document_from_template()           │
   │ Заменяет {{ключ}} на значения               │
   │ Создает файл Word/Excel/PowerPoint          │
   └─────────────────────────────────────────────┘
                     ↓
   ┌─────────────────────────────────────────────┐
   │ Document.generated_file = путь к файлу      │
   │ Пользователь скачивает готовый документ     │
   └─────────────────────────────────────────────┘
```

## Ключевые изменения

### Было (старая система):
```python
# Жестко заданные поля
class DocumentFromTemplateForm(forms.Form):
    full_name = forms.CharField(...)
    position = forms.CharField(...)
    department = forms.CharField(...)
    custom_field_1 = forms.CharField(...)
    custom_field_2 = forms.CharField(...)

# Жестко заданные замены
replacements = {
    'имя': form.cleaned_data['full_name'],
    'должность': form.cleaned_data['position'],
    'отдел': form.cleaned_data['department'],
}
```

❌ **Проблемы:**
- Нельзя добавить новые поля без изменения кода
- Все шаблоны имеют одинаковые поля
- Неиспользуемые поля отображаются всегда
- Нет гибкости

### Стало (новая система):
```python
# Динамическое создание полей
class DocumentFromTemplateForm(forms.Form):
    def __init__(self, *args, template_id=None, **kwargs):
        super().__init__(*args, **kwargs)
        if template_id:
            template = DocumentTemplate.objects.get(pk=template_id)
            for key, config in template.placeholders.items():
                field = create_field_by_type(config)
                self.fields[f'placeholder_{key}'] = field

# Динамические замены
replacements = {}
for key in template.placeholders.keys():
    value = form.cleaned_data.get(f'placeholder_{key}', '')
    replacements[key] = str(value)
```

✅ **Преимущества:**
- Любые поля на выбор админа
- Каждый шаблон со своим набором полей
- Только нужные поля отображаются
- Полная гибкость

## Известные ограничения

1. **Перезагрузка страницы при выборе шаблона**
   - При изменении шаблона страница перезагружается
   - Несохраненные данные теряются
   - Возможно улучшение через AJAX в будущем

2. **Ограниченные типы полей**
   - Поддерживаются: text, textarea, date, number
   - Не поддерживаются: select, checkbox, radio, file
   - Возможно расширение в будущих версиях

3. **Нет группировки полей**
   - Все поля отображаются списком
   - Нет разделения на секции/категории
   - Возможно добавление в будущем

## Обратная совместимость

✅ **Старые шаблоны продолжают работать:**
- Шаблоны без placeholders поля будут работать
- Форма покажет только стандартные поля
- Генерация документов без изменений

✅ **Миграция БД не требуется:**
- Поле placeholders уже существует (JSONField)
- Новые записи просто заполняют это поле
- Старые записи имеют NULL или пустой JSON
