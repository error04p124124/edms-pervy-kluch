# Система электронного документооборота для ООО "Первый ключ"

Веб-приложение на Django 5.x для автоматизации делопроизводства и управления документами.

## 🎯 Основные возможности

### Для делопроизводителя (главная роль):
- ✅ Создание документов из готовых шаблонов с автоподстановкой данных
- ✅ Автоматическая регистрация и присвоение уникальных номеров (формат: ГГГГ/ММ/НННН)
- ✅ Полный реестр документов с мощными фильтрами
- ✅ Управление маршрутами согласования
- ✅ Массовые операции (архивирование, отправка на согласование)
- ✅ Мониторинг сроков и подсветка просроченных документов
- ✅ Отчеты и аналитика
- ✅ Управление шаблонами документов

### Для руководителей:
- ✅ Просмотр документов на согласовании
- ✅ Утверждение/отклонение с комментариями
- ✅ Уведомления о новых документах

### Для сотрудников:
- ✅ Создание документов и заявок
- ✅ Отслеживание статуса своих документов
- ✅ Уведомления об изменениях

## 🛠 Технологии

- **Backend:** Django 5.1.4, Python 3.x
- **Database:** SQLite (по умолчанию)
- **Frontend:** Bootstrap 5, Font Awesome, Chart.js
- **Forms:** django-crispy-forms с Bootstrap 5
- **PDF Export:** WeasyPrint
- **Config:** python-dotenv

## 📋 Требования

- Python 3.10+
- pip
- virtualenv (рекомендуется)

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
cd d:\12112
```

### 2. Создание виртуального окружения

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Linux/Mac
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения

Скопируйте `.env.example` в `.env` и настройте:

```bash
copy .env.example .env  # Windows
cp .env.example .env    # Linux/Mac
```

Отредактируйте `.env`:
```
SECRET_KEY=ваш-секретный-ключ-для-production
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1
```

### 5. Применение миграций

```bash
python manage.py makemigrations
python manage.py migrate
```

### 6. Создание суперпользователя

```bash
python manage.py createsuperuser
```

Введите данные администратора:
- Username (имя пользователя)
- Email
- Password

### 7. Создание директорий для медиа-файлов

```bash
mkdir media
mkdir media\documents
mkdir media\avatars
```

### 8. Загрузка статических файлов (опционально)

```bash
python manage.py collectstatic --noinput
```

### 9. Запуск сервера разработки

```bash
python manage.py runserver
```

Откройте браузер: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## 👤 Настройка ролей пользователей

### Через админ-панель Django

1. Перейдите в админ-панель: [http://127.0.0.1:8000/admin/](http://127.0.0.1:8000/admin/)
2. Войдите под учетной записью суперпользователя
3. Перейдите в раздел "Users"
4. Выберите пользователя и в разделе "Профиль" установите:
   - **Роль:** clerk (делопроизводитель), manager (руководитель), employee (сотрудник), admin (администратор)
   - **Отдел:** название отдела
   - **Должность:** должность сотрудника

### Роли в системе:

- **Делопроизводитель (clerk):** Полный доступ ко всем функциям
- **Руководитель (manager):** Согласование и утверждение документов
- **Сотрудник (employee):** Создание и просмотр своих документов
- **Администратор (admin):** Управление пользователями и настройками

## 📝 Создание первых шаблонов

### Пример шаблона приказа:

```html
<h2>ПРИКАЗ №{{номер}}</h2>
<p>от {{дата}}</p>

<h3>О приеме на работу</h3>

<p>Приказываю принять на должность {{должность}} в {{отдел}} сотрудника {{имя}}.</p>

<p>Дата начала работы: {{custom1}}</p>
<p>Оклад: {{custom2}}</p>

<p>Основание: Заявление сотрудника, Трудовой договор</p>

<br><br>
<p>Руководитель ________________</p>
```

### Доступные плейсхолдеры:

- `{{имя}}` - ФИО сотрудника
- `{{должность}}` - Должность
- `{{отдел}}` - Название отдела
- `{{дата}}` - Текущая дата (подставляется автоматически)
- `{{custom1}}`, `{{custom2}}` - Дополнительные поля

## 📊 Основные модели данных

### UserProfile
- Расширение стандартного User
- Роль, отдел, должность, аватар

### DocumentTemplate
- Шаблоны для создания документов
- Тип документа, HTML-шаблон с плейсхолдерами

### Document
- Основная модель документа
- Автоматическая нумерация, статусы, сроки
- Маршрутизация согласования

### DocumentHistory
- История всех изменений документа
- Аудит действий пользователей

### Notification
- Система уведомлений пользователей

### WorkflowStep
- Этапы маршрута согласования

## 🐛 Решение проблем

### Ошибка при установке WeasyPrint

WeasyPrint требует дополнительных системных библиотек:

**Windows:**
- Скачайте GTK3 runtime: https://github.com/tschoonj/GTK-for-Windows-Runtime-Environment-Installer/releases

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get install python3-dev python3-pip python3-cffi libcairo2 libpango-1.0-0 libpangocairo-1.0-0 libgdk-pixbuf2.0-0 libffi-dev shared-mime-info
```

**Mac:**
```bash
brew install cairo pango gdk-pixbuf libffi
```

### База данных заблокирована

Убедитесь, что не запущено несколько экземпляров сервера разработки.

### Статические файлы не загружаются

```bash
python manage.py collectstatic --clear --noinput
```
